🔎 The Five Blockers — All Resolved

Found in the code. Fixed in the code. Proven by a test. Not on a spreadsheet.

1

Hardcoded Auth Service IP — http://10.0.1.45:8080/auth/validate

Not routable in Azure. Every authenticated request would timeout silently after 30 seconds. Fixed: AUTH_SERVICE_URL env var takes priority; default is documented not hardcoded.

✓ Fixed & Tested
2

Local Filesystem Log File — /var/log/contoso/app.log

Container Apps ephemeral filesystem discards all logs on restart. Fixed: LOG_FILE defaults to null (stdout). Logs go to Azure Monitor automatically.

✓ Fixed & Tested
3

NFS-Mounted Feed Directory — /mnt/findata/feeds

Batch job would silently process zero rows and exit success — Finance ops wouldn't notice until GL entries were missing. Fixed: raises EnvironmentError if neither Blob Storage nor local path is set. Loud failure, not silent.

✓ Fixed & Tested
4

Plaintext DB Password in /etc/contoso/config.ini

Password unchanged since 2021; three former employees had access. Fixed: env vars only. Azure Key Vault secret reference in Terraform. Managed Identity for app-to-DB auth.

✓ Fixed & Tested
5

Redis Keepalive Cron — hits http://10.0.1.30:9090/cache/warm

Undocumented cron on app01.internal. Not routable in Azure. Azure Cache for Redis manages eviction natively — no keepalive needed. Cron eliminated entirely.

✓ Fixed & Tested

7 Risks the Human Discovery Missed

!

Split-brain cutover on transactions table — must not migrate separately

Batch writes UPDATE transactions SET status='reconciled'. Webapp reads that status for customers. Migrating in separate windows leaves customers seeing stale data while the databases diverge. Discovery noted both workloads use PostgreSQL; missed the shared mutable write.

Critical
!

Redis TLS port change (6379 → 6380) breaks webapp sessions at startup

Azure Cache for Redis mandates TLS on 6380. Discovery found the keepalive cron (Finding #5) but not the webapp's own Redis client. Without REDIS_TLS=true and REDIS_PORT=6380, the webapp crashes at boot.

Critical
!

X-PII-Scope: internal header is spoofable from the internet

Any caller can send this header and receive fully unredacted PII (email, phone, account number, balance). On-prem, the network perimeter blocked this. Behind a public Container Apps ingress, it's an open door. Must be gated before go-live.

Critical
!

SSL defaults are wrong for both workloads

Webapp: DB_SSL=false. Batch: DB_SSLMODE=prefer. Azure PostgreSQL Flexible Server enforces TLS — both will be refused at runtime unless SSL is explicitly configured.

Critical
!

PostgreSQL version discrepancy: on-prem 14 → target 16

pg_dump from PG14 into PG16 needs extension version validation (pgaudit, pgcrypto) and type-cast review. Not acknowledged in the discovery document.

Action required
!

No GDPR Article 17 right-to-erasure mechanism

ON DELETE RESTRICT foreign keys will block any attempt to delete a customer record. Must be resolved before migrated data is live in Azure UK South.

Action required
!

Connection pool exhaustion across all three workloads

Webapp (10 conns) + batch (burst at 02:00) + five internal BI/Risk/Finance/Ops teams, all on one Flexible Server SKU. Risk only visible when all three budgets are summed. PgBouncer is a Phase 2 prerequisite, not a nice-to-have.

Phase 2
Migration order the coordinator recommends: reporting-db first (all three depend on it) → then batch + webapp in the same maintenance window (02:00–04:00 UTC) to avoid split-brain. Separate cutover windows for the two application workloads is a data-correctness risk, not just an inconvenience.

PII Controls

Private endpoints everywhere

PII in API Responses

Redacted by default

PCI-DSS (Reporting DB)

Secret Management

Audit Trail

Structured JSON logs throughout

Both workloads emit structured JSON to stdout. No PII in any log line. Azure Monitor collects via Container Apps log stream.

Before Cutover (Q3 2026)

1

Fix SSL defaults in both workloads

Set DB_SSL=true (webapp) and DB_SSLMODE=require (batch). Non-negotiable for Azure PostgreSQL.

2

Set Redis TLS config on webapp

REDIS_TLS=true and REDIS_PORT=6380 in Container App environment variables.

3

Gate X-PII-Scope at Container Apps ingress

Block the header at the ingress policy layer — not just in application code.

4

Rehearse rollback runbook (Challenge 8)

DNS revert, point-in-time restore, batch re-enable. SRE sign-off required before cutover window.

Phase 2 — H2 2026

🔐
